================================================================================
                          OOP CONCEPTS DEMONSTRATED
================================================================================

┌─────────────────────┬──────────────────────────────────────────────────────┐
│ CONCEPT             │ WHERE IT'S DEMONSTRATED                              │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ ENCAPSULATION       │ • Private fields (_title, _options, _correctAnswer)  │
│                     │ • Public properties with controlled access          │
│                     │ • Validation in setters/Validate() methods          │
│                     │ • Private methods in classes                        │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ INHERITANCE         │ • BaseEntity → Quiz, Question                       │
│                     │ • Question → MultipleChoiceQuestion, etc.            │
│                     │ • BaseService → Service classes (optional)           │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ POLYMORPHISM        │ • IQuestionType interface with multiple             │
│                     │   implementations                                    │
│                     │ • Method overriding (GetDisplayName, Validate)      │
│                     │ • Factory pattern returns interface                  │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ ABSTRACTION         │ • BaseEntity abstract class                         │
│                     │ • IQuestionType, IQuizService interfaces             │
│                     │ • Abstract methods force implementation              │
│                     │ • Virtual methods allow overriding                  │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ COHESION            │ • Each class has single, focused responsibility     │
│                     │ • Related methods grouped together                   │
│                     │ • Easy to understand and maintain                   │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ LOW COUPLING        │ • Dependencies on interfaces, not concrete classes  │
│                     │ • Services depend on DbContextFactory abstraction    │
│                     │ • Easy to test with mocks                            │
│                     │ • Flexible and extensible                           │
└─────────────────────┴──────────────────────────────────────────────────────┘

================================================================================
                          LEGEND
================================================================================

+ = Public member
- = Private member
# = Protected member
▲ = Inheritance (is-a relationship)
│ = Implements (interface implementation)
─ = Association (has-a relationship)
<*> = One-to-many relationship
(1) = One
(*) = Many
